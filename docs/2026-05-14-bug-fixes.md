# Bug 修复记录 — 2026-05-14

## 1. 编辑头像后过一会自动变成昵称首字母

### 问题
在个人资料页编辑头像后，头像正常显示一段时间，随后自动变回首字母 fallback。

### 根因
- `profile_page.dart` 中头像显示使用 `CachedNetworkImage`，其 `placeholder` 和 `errorWidget` 都设置为昵称首字母
- Supabase `onAuthStateChange` 会周期性触发（JWT token 刷新），调用 `notifyListeners()` 导致页面 rebuild
- rebuild 时 `CachedNetworkImage` 需要重新加载网络图片，若加载慢或失败，就显示首字母 fallback

### 修复
- **`lib/providers/auth_provider.dart`**：上传头像后复制到本地 `avatar_cache` 文件；`_loadProfile` 时后台下载头像缓存到本地
- **`lib/pages/profile_page.dart`**：显示优先级改为：上传中临时文件 → 本地缓存文件 → 网络 `CachedNetworkImage` → 首字母 fallback

## 2. 分享非 JPG 格式图片失败

### 问题
通过好友分享功能分享心情时，JPG 图片可以正常显示，但 PNG/GIF/WebP 等格式图片分享后无法查看。

### 根因
- `image_picker` 设置 `imageQuality: 80` 后，iOS/Android 会将图片内容重新编码为 JPEG，但 `xFile.name` 可能保留原始扩展名（如 `.png`）
- 文件被保存为 `xxx.png`，实际内容是 JPEG
- Supabase SDK 的 `storage_upload` 根据文件扩展名推断 `Content-Type`（`.png` → `image/png`）
- 接收方下载时按 PNG 解码 JPEG 数据 → 失败

### 修复
- **`lib/services/image_manager.dart`**：在 `saveImageToFile` 中新增 `_detectExtension` 方法，通过读取文件头魔数（magic bytes）判断实际图片格式，使用正确的扩展名保存

## 4. 点击收件箱时短暂闪现红色错误页

### 问题
每次切换到收件箱 tab 时，会短暂闪现一个红色错误页面（带英文堆栈信息），很快消失。

### 根因
- `home_page.dart` 使用 `_pages[_selectedIndex]` 切换 tab，每次切换都会**销毁旧页面、重新创建新页面**
- `MessagePage.initState` 中 `_ensureBoxesOpen()` 是异步操作（打开 Hive box）
- 第一次 `build` 在 box 打开**之前**执行，`_buildAiMessagesSection` 访问未初始化的 `late` 变量 `_mailBox` → `LateInitializationError`
- 几十毫秒后 box 打开完成 → `setState` → 页面恢复正常

### 修复
- **`lib/pages/message_page.dart`**：
  - `_mailBox` / `_logBox` 从 `late` 改为可空类型
  - 新增 `_isBoxReady` 标志，box 打开完成后设为 `true`
  - `_buildAiMessagesSection` 检查 `_isBoxReady`，未就绪时显示 loading
  - 所有访问 `_mailBox` / `_logBox` 的方法增加 null guard
