# 01-万物皆对象与变量

## C++ → Dart 心智切换

> **一句话：** C++ 里 `int` 是值、`std::string` 是对象；Dart 里**一切皆对象**，包括 `int`、`bool`、`null` 都是对象，所有值都是引用语义。

| C++ | Dart |
|-----|------|
| 值类型 vs 引用类型严格区分 | 一切皆对象，没有值类型 |
| `int x = 1;` 栈上分配 | `int x = 1;` 堆上分配（但不可变对象共享） |
| `const int x = 1;` 运行时常量 | `const x = 1;` **编译期常量**（完全不同！） |
| `auto x = 1;` 类型推导 | `var x = 1;` 类型推导，但一旦赋值类型就锁定 |
| `nullptr` / `NULL` | `null`（是一个 Null 对象的引用） |
| 无 null safety | **强制 null safety**（编译器级别） |

## 一、var / final / const / dynamic

这四个是你写 Dart 每行代码都会遇到的。**关键区别：**

```dart
// var: 类型推导，一旦赋值类型就锁定
var x = 1;        // x 是 int，不能再 x = "hello"
var y;            // y 是 dynamic（没有初始值无法推导）

// final: 运行时常量，只能赋值一次（类似 C++ 的 const，但更严格）
final now = DateTime.now();  // 编译时不知道值，运行时确定
final z;          // ✅ 可以先声明后赋值一次

// const: 编译期常量（C++ 没有直接对应物，比 constexpr 更严格）
const pi = 3.14;            // 编译时就确定了
const list = [1, 2, 3];     // 整个列表编译期不可变
// const bad = DateTime.now(); // ❌ 编译报错！运行时才知道值

// dynamic: 关闭类型检查（类似把 C++ 当 Python 写）
dynamic anything = 1;
anything = "hello";  // ✅ 可以
anything.foo();      // 编译不报错，运行时可能崩
```

### 在你项目里的真实用例

打开 `lib/models/mood_record.dart:4-17`：

```dart
class MoodRecord {
  final String id;          // final: 每个实例的 id 在构造时确定，此后不可变
  final String ownerId;
  final String? localId;    // String? 是可空类型，后面模块会讲
  final String moodType;
  final String note;
  final String comment;
  final List<String> imageUrls;
  final String? audioUrl;
  final int? audioDuration;
  final String? customEmoji;
  final String? customEmojiLabel;
  final int? customColorValue;
  final String? aiComfort;
  final bool aiEnabled;
  final DateTime createdAt;
```

**为什么都用 `final` 而不是 `const`？** 因为这些值在编译时不知道，是运行时从数据库/网络获取的。C++ 里你写 `const std::string id` 也可以，但 Dart 的 `const` 要求编译期已知，所以只能用 `final`。

## 二、类型系统：强类型 + 类型推导

Dart 是**强类型**语言（和 C++ 一样），但类型声明可以省略，交给推导：

```dart
// 三种等价写法
int count = 1;          // 显式类型（像 C++ int count = 1;）
var count = 1;          // 推导为 int
final count = 1;        // 推导为 int 且不可变

// Dart 社区推荐：局部变量用 var/final，公开接口显式写类型
```

看 `lib/services/ai_service.dart:196-200`：

```dart
static Future<String> _postWithRetry(   // 返回类型显式声明
  String url,                            // 参数类型显式声明
  Map<String, String> headers,           // 泛型！类似 std::map<string, string>
  Map<String, dynamic> body,             // dynamic 当值类型 = JSON 的万能容器
) async {
```

这里 `Map<String, dynamic>` 是 Dart 里处理 JSON 的标配写法——因为 JSON 的值可以是 string/int/bool/null/嵌套，用 `dynamic` 关掉类型检查。

## 三、内置类型速览

```dart
// 数字（比 C++ 简单太多，没有 short/long/unsigned/float/double 的大杂烩）
int x = 1;           // 任意精度整数（没有 int32/int64 之分，概念上类似 Python int）
double y = 3.14;     // 64-bit 浮点
num z = 1;           // num 是 int 和 double 的基类
z = 3.14;            // ✅ num 可以同时接收 int 和 double

// 字符串
String s = 'hello';         // 单引号
String s2 = "hello";        // 双引号都可以
String s3 = 'hello $s';     // $变量名 字符串插值（C++ 没有，类似 fmt::format）
String s4 = '1 + 1 = ${1 + 1}';  // ${表达式}
// 多行字符串（三引号）
String multi = '''
第一行
第二行
''';

// 布尔
bool flag = true;   // 只有 true/false，没有 0/1 的隐式转换

// List（就是 C++ 的 std::vector，但更灵活）
List<int> nums = [1, 2, 3];
var nums2 = <int>[1, 2, 3];  // 泛型推导

// Set
Set<String> tags = {'开心', '难过'};

// Map（就是 C++ 的 std::map / std::unordered_map）
Map<String, int> scores = {'语文': 90, '数学': 85};
```

## 四、C++ 程序员最容易踩的坑

### 坑 1：const 的含义完全不同

```cpp
// C++: const 表示"我不修改这个变量"
const int x = 1;
const std::string& s = getString();  // 不拷贝，不修改
```

```dart
// Dart: const 表示"编译期就知道，且完全不可变（包括深层）"
const list = [1, 2, 3];
// list.add(4);  // ❌ 编译报错
// 而 C++ 的 const vector 只是不能改引用，const_iterator 只是不能改元素
```

**Dart 的 const 是"规范化的、编译期完全确定的、深度不可变的"**——更接近 C++20 的 `consteval` + 深层 const。

### 坑 2：所有东西都是引用

```dart
var a = [1, 2, 3];
var b = a;       // b 和 a 指向同一个 List 对象
b.add(4);
print(a);        // [1, 2, 3, 4] —— a 也被改了！
```

C++ 里 `auto b = a;` 会拷贝 `std::vector`，Dart 不会——和 Java/Python 一样是引用语义。

### 坑 3：没有函数重载

```cpp
// C++ ✅
void foo(int x);
void foo(double x);
void foo(int x, int y);
```

```dart
// Dart ❌ 同一个作用域内函数名必须唯一
// 替代方案：命名参数（模块 03 会讲）
void foo(int x) {}
void foo(double x) {}  // ❌ 编译报错：'foo' is already defined
```
