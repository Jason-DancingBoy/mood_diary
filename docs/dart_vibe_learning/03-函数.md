# 03-函数

## C++ → Dart 心智切换

> **一句话：** C++ 函数靠重载区分签名，Dart 没有函数重载，靠**命名参数**实现同样的表达能力。

| C++ | Dart |
|-----|------|
| `void foo(int x, int y=0)` 默认参数 | `void foo(int x, {int y = 0})` 命名可选参数 |
| `void foo(int x); void foo(double x);` 重载 | ❌ 不支持重载，靠不同函数名或命名参数区分 |
| `auto f = [](int x){ return x*2; };` lambda | `var f = (int x) => x * 2;` 箭头函数 |
| `std::function<int(int)>` | `int Function(int)` 类型声明 |
| `void bar(std::function<void()> cb)` 回调 | `void bar(void Function() cb)` |
| 函数指针 / `&ClassName::method` | tear-off: `ClassName.method` (无需 & ) |

## 一、参数传递：Dart 没有函数重载，怎么办？

C++ 里你习惯这样写：

```cpp
void log(const std::string& msg);
void log(const std::string& msg, int level);
void log(const std::string& msg, int level, bool toFile);
```

Dart **同一个作用域内函数名必须唯一**，替代方案是命名参数：

```dart
// 命名参数用 {} 包裹，调用时必须写参数名
void log(String msg, {int level = 0, bool toFile = false}) {
  // ...
}

log("服务器启动了");                        // 只传必填的
log("警告", level: 1);                     // 可选参数用 参数名: 值
log("错误", level: 2, toFile: true);       // 顺序任意
```

### 参数类型三件套

```dart
// 1. 必填位置参数 — 调用时必须按顺序提供
void foo(int x, String y) {}

// 2. 可选位置参数 — 用 [] 包裹，可以不传
void bar(int x, [String y = 'default', int? z]) {}
bar(1);           // ✅ y 用默认值，z 为 null
bar(1, 'hello');  // ✅
bar(1, 'hello', 3); // ✅

// 3. 命名参数 — 用 {} 包裹
//    - required 前缀 = 必填
//    - 有默认值 = 可选
//    - 无默认值且无 required = 可选，默认 null
void baz({
  required int x,        // 必须传
  String y = 'hello',    // 可选，有默认值
  int? z,                // 可选，默认 null
}) {}

baz(x: 1);                    // ✅ 只传必填的
baz(x: 1, z: 3);              // ✅ 命名参数顺序任意
baz(y: 'hi', x: 2);           // ✅ 顺序无所谓
// baz();                      // ❌ 缺少 required 参数 x
```

## 二、箭头函数（Arrow Function）

Dart 的 `=>` 就是 C++ lambda 的简洁版，**但只能写一个表达式**（不能写语句块）：

```dart
// 单行函数体可以用 =>
int add(int a, int b) => a + b;

// 等价于
int add(int a, int b) {
  return a + b;
}

// Lambda 赋值给变量（Dart 中函数是一等公民）
var multiply = (int a, int b) => a * b;
print(multiply(3, 4));  // 12

// 作为回调传递（类似 C++ 的 lambda）
list.map((item) => item.toUpperCase());
list.where((item) => item.length > 3);
list.forEach(print);  // tear-off，后面讲
```

项目里的例子——`lib/models/ai_assistant.dart` 和 `lib/services/supabase_service.dart:15-16`：

```dart
// getter 里用箭头函数
Map<String, String> toMap() => {'role': role, 'content': content};

// 单例 getter 用箭头函数
static PostgrestQueryBuilder get profiles =>
    Supabase.instance.client.from('profiles');
```

## 三、闭包（Closure）

和 C++ lambda 的捕获一样，Dart 函数天然是闭包——自动捕获外部变量：

```dart
Function makeCounter() {
  int count = 0;           // 局部变量
  return () {              // C++: return [&count]() { return ++count; };
    count++;
    return count;
  };
}

var counter = makeCounter();
print(counter());  // 1
print(counter());  // 2
print(counter());  // 3
// count 变量"活"在了返回的函数里
```

注意：Dart 没有 C++ 的 `[=]` / `[&]` 区分，闭包捕获的都是**变量的引用**，不是值的拷贝。在 for 循环里写闭包要特别小心：

```dart
// ❌ 常见坑：所有闭包捕获的是同一个变量 i
var funcs = <Function>[];
for (var i = 0; i < 3; i++) {
  funcs.add(() => print(i));
}
funcs.forEach((f) => f());  // 输出 3, 3, 3 —— 不是 0, 1, 2!

// ✅ 修复：在每次迭代中创建一个新变量
for (var i = 0; i < 3; i++) {
  var captured = i;          // 每次迭代都是新的 captured
  funcs.add(() => print(captured));
}
funcs.forEach((f) => f());  // 输出 0, 1, 2 ✅
```

## 四、Tear-off（函数引用）

C++ 里你要传成员函数，得写 `&ClassName::method`。Dart 更简单：

```dart
// C++: std::for_each(v.begin(), v.end(), print);
// Dart: 直接写函数名，不加括号
list.forEach(print);  // 等价于 list.forEach((item) => print(item));

// 实例方法也一样
class Foo {
  void bar(String s) => print(s);
}
var foo = Foo();
var f = foo.bar;  // tear-off：把 foo 的 bar 方法"撕下来"当函数用
f('hello');       // 等价于 foo.bar('hello');

// 常见用例
[1, 2, 3].map(square);       // 传顶层函数引用
stream.listen(handleData);    // 传回调
Future.delayed(dur, callback); // 同上
```

## 五、函数类型声明

Dart 可以直接用函数签名当类型：

```dart
// 类型声明：接收 int，返回 String
String Function(int) toString;

// 可以配合 typedef 使语义更清晰
typedef IntMapper = int Function(int);
IntMapper doubler = (int x) => x * 2;

// typedef 也可以用于回调
typedef OnDataCallback = void Function(String data);
void fetchData(OnDataCallback onData) {
  onData('got it');
}
```

## 六、C++ 程序员最容易踩的坑

### 坑 1：没有函数重载 → 命名参数是唯一出路

```cpp
// C++ ✅
void setConfig(int port);
void setConfig(String host, int port);
```

```dart
// Dart ❌ 报错：'setConfig' is already defined
// void setConfig(int port) {}
// void setConfig(String host, int port) {}

// Dart ✅ 用命名参数整合
void setConfig({String? host, int? port}) {}
```

### 坑 2：闭包捕获的是引用，不是值

前面 for 循环的例子。C++ 里 `[=]` 默认拷贝，Dart 不存在这种语义。

### 坑 3：箭头函数 ≠ 多语句函数

```dart
// ✅ 单个表达式
int square(int x) => x * x;

// ❌ 不能写语句块
// int foo(int x) => { print(x); return x; };  // 编译报错

// ✅ 多语句用普通函数体
int foo(int x) {
  print(x);
  return x;
}
```
