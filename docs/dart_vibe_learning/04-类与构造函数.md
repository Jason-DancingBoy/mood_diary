# 04-类与构造函数

## C++ → Dart 心智切换

> **一句话：** C++ 一个类可以有多个构造函数靠重载，Dart **没有构造重载**，靠"命名构造函数"和"factory 构造函数"实现同样的灵活性。

| C++ | Dart |
|-----|------|
| `ClassName(int x) : _x(x) {}` 初始化列表 | `ClassName(this.x);` 语法糖（初始化形参） |
| 构造函数重载 | 命名构造函数 `ClassName.fromMap(...)` |
| 拷贝构造 / 移动构造 | ❌ 无，所有对象引用语义 |
| 析构函数 `~ClassName()` | ❌ 无析构（GC 管理） |
| `virtual void foo() = 0;` 纯虚函数 | `abstract` 类 / 隐式接口 |
| `class Derived : public Base` | `class Derived extends Base` 单继承 |
| 多重继承 `class C : public A, public B` | ❌ 单继承，用 mixin 替代（模块 05） |
| `const` 成员函数 | 无直接对应，`const` 构造函数用于编译期常量对象 |

## 一、构造函数全家桶

### 1. 最简构造函数

```dart
class Point {
  int x;
  int y;

  // 最简写法：自动把参数赋给同名字段
  Point(this.x, this.y);
}

var p = Point(1, 2);
```

`this.x` 在参数列表里是 Dart 特有的语法糖，C++ 没有——自动等价于 `this.x = x`。

### 2. 命名构造函数

因为没有重载，Dart 用 `ClassName.xxx()` 格式创建多种构造方式：

```dart
class Point {
  int x, y;
  Point(this.x, this.y);

  // 命名构造函数：从原点创建
  Point.origin()
      : x = 0,
        y = 0;

  // 命名构造函数：从 JSON 创建
  Point.fromJson(Map<String, int> json)
      : x = json['x'] ?? 0,
        y = json['y'] ?? 0;
}

var a = Point(1, 2);              // 默认构造
var b = Point.origin();           // 命名构造
var c = Point.fromJson({'x': 3}); // 命名构造
```

### 3. 初始化列表（Initializer List）

`:` 后面跟初始化列表，在构造函数体执行**之前**运行：

```dart
class Rectangle {
  final int width;
  final int height;

  // 初始化列表可以直接赋值给 final 字段
  Rectangle(int size)
      : width = size,
        height = size;

  // 初始化列表可以包含断言
  Rectangle.fromSize(this.width, this.height)
      : assert(width > 0),
        assert(height > 0);
}

// C++ 对应:
// Rectangle(int size) : width(size), height(size) {}
// 语法很像，但 Dart 里 final 字段必须在初始化列表或声明处赋值
```

### 4. 初始化形参（Initializing Formal）

`this.xxx` 放在参数列表里是最常见的写法，因为省代码：

```dart
class User {
  final String name;
  final int age;

  // 三个等价的写法：
  // 写法1: this 语法糖（最常用）
  User(this.name, this.age);

  // 写法2: 初始化列表（等价但啰嗦）
  // User(String name, int age)
  //     : name = name,
  //       age = age;

  // 写法3: 构造函数体赋值（final 字段不能这样写！）
  // User(String name, int age) {
  //   this.name = name;  // ❌ final 字段不能在构造体内赋值
  // }
}
```

MoodRecord({

## 二、Factory 构造函数

Dart 独有的概念——`factory` 构造函数**不一定创建新对象**（可能返回缓存的、子类的、或已有实例），而普通构造函数**总是创建新实例**。

```dart
class Logger {
  final String name;
  static final Map<String, Logger> _cache = {};

  // factory 构造函数：实现对象缓存（单例模式）
  factory Logger(String name) {
    return _cache.putIfAbsent(name, () => Logger._internal(name));
  }

  // 私有命名构造函数（真正的构造逻辑）
  Logger._internal(this.name);
}

var a = Logger('app');
var b = Logger('app');
print(identical(a, b));  // true —— 返回同一个对象！
```

项目里 `factory` 最常见用于"从 Map/List 反序列化"：

`lib/models/mood_record.dart:38-58`：

```dart
factory MoodRecord.fromMap(Map<String, dynamic> map) {
  return MoodRecord(
    id: map['id'] as String,
    ownerId: map['owner_id'] as String,
    // ...
  );
}
```

`lib/models/chat_message.dart:84-98`（另一个 factory fromMap）：

```dart
factory ChatMessage.fromMap(Map<String, dynamic> map, String currentUserId) {
  final isAi = map['is_ai_message'] as bool? ?? false;
  final isUser = isAi ? false : map['sender_id'] == currentUserId;
  return ChatMessage(
    isUser: isUser,
    id: map['id'] as String?,
    content: map['content'] as String? ?? '',
    // ...
  );
}
```

Factory vs 普通构造的区别：

| | 普通构造函数 | factory 构造函数 |
|---|---|---|
| 一定创建新对象 | ✅ | ❌（可能返回已有对象） |
| 可以使用 `this` | ✅ | ❌ |
| 可以返回子类实例 | ❌ | ✅ |
| 典型用途 | 创建对象 | 反序列化、缓存、单例 |

## 三、继承

Dart 是**单继承**（不像 C++ 多继承），用 `extends`：

```dart
// 基类
class Animal {
  final String name;
  Animal(this.name);

  void speak() => print('...');
}

// 派生类
class Dog extends Animal {
  final String breed;

  // 调用父类构造函数用 super
  Dog(String name, this.breed) : super(name);

  // 覆盖父类方法用 @override（推荐写，不强制）
  @override
  void speak() => print('汪汪！');
}
```

项目里的实例——`lib/providers/auth_provider.dart:9`：

```dart
class AuthProvider extends ChangeNotifier {
  User? _user;
  UserProfile? _profile;

  AuthProvider() {
    _user = AuthService.currentUser;
    AuthService.onAuthChange.listen(_onAuthStateChanged);
    // ...
  }
```

`ChangeNotifier` 是 Flutter 状态管理的核心基类，`AuthProvider` 继承它来实现响应式通知。`super` 参数在 Dart 里也有语法糖：

```dart
class Dog extends Animal {
  final String breed;
  // super.name 直接把参数透传给父类构造器
  Dog(super.name, this.breed);
  //         ↑ 等价于 Dog(String name, this.breed) : super(name);
}
```

## 四、抽象类和接口

```dart
// 抽象类：不能实例化，可以有抽象方法
abstract class Shape {
  double get area;              // 抽象 getter（没有实现体）
  double get perimeter;         // 抽象 getter

  void log() => print('面积：$area');  // 抽象类也可以有具体方法
}

class Circle extends Shape {
  final double radius;
  Circle(this.radius);

  @override
  double get area => 3.14 * radius * radius;

  @override
  double get perimeter => 2 * 3.14 * radius;
}

// Shape s = Shape();  // ❌ 抽象类不能实例化
Shape s = Circle(5);   // ✅ 可以指向子类实例
```

Dart 里任何类都可以隐式作为接口被 `implements`（不需要单独定义 interface），这点比 C++ 灵活：

```dart
class Flyer {
  void fly() => print('飞');
}

// implements 意思是"我承诺提供 Flyer 声明的所有方法"
// 和 extends 不同：不会继承实现，必须全部自己写
class Bird implements Flyer {
  @override
  void fly() => print('鸟在飞');
}
```

(模块 05 会讲 `mixin`，它是 C++ 多继承在 Dart 里的替代，比 `implements` 更强大)

## 五、Getter / Setter

Dart 的 getter / setter 本质上是用方法语法模拟属性访问，比 C++ 简洁得多：

```dart
class Temperature {
  double _celsius;  // _ 前缀 = 私有（类似 C++ private）

  Temperature(this._celsius);

  // getter: 访问时像属性，实际是计算值
  double get fahrenheit => _celsius * 9 / 5 + 32;

  // setter: 赋值时触发逻辑
  set fahrenheit(double value) {
    _celsius = (value - 32) * 5 / 9;
  }
}

var t = Temperature(0);
print(t.fahrenheit);    // 32.0  ← 像读属性，实为调用 getter
t.fahrenheit = 212;     //        ← 像写属性，实为调用 setter
print(t._celsius);      // 100.0
```

项目里的 getter——`lib/providers/auth_provider.dart:18-24`：

```dart
User? get user => _user;
UserProfile? get profile => _profile;
bool get isLoggedIn => _user != null;
bool get isLoading => _isLoading;
String? get error => _error;
```

`isLoggedIn` 暴露为属性，外部用 `if (authProvider.isLoggedIn)` 比 `if (authProvider.isLoggedIn())` 更自然。

chat_message.dart 也有 getter：

```dart
bool get isVoiceMessage => audioUrl != null;
```

## 六、const 构造函数

模块 01 说了 Dart 的 `const` 是编译期常量。类也可以有 `const` 构造函数——所有字段都是 `final`，构造出的对象在编译期确定：

```dart
class Color {
  final int r, g, b;
  const Color(this.r, this.g, this.b);
}

const red = Color(255, 0, 0);    // 编译期常量
var red2 = Color(255, 0, 0);     // 运行时对象（值相同但是不同实例）

print(identical(const Color(255,0,0), const Color(255,0,0)));  // true！同一个对象
print(identical(Color(255,0,0), Color(255,0,0)));              // false！不同实例
```

这就是模块 01 提到的"规范化"（canonicalized）——相同的 `const` 对象在内存中只有一份。

## 七、C++ 程序员最容易踩的坑

### 坑 1：没有析构函数

```cpp
// C++: RAII —— 析构函数自动释放资源
class FileHandle {
  ~FileHandle() { close(fd); }
};
```

Dart **没有析构函数**。对象由 GC 回收，你不控制回收时机。资源释放要手动调用 `dispose()`：

```dart
class MyWidget {
  final controller = StreamController();

  // ⚠️ Dart 不会自动调这个，你必须手动调用
  void dispose() {
    controller.close();
  }
}
```

### 坑 2：所有字段赋值都在构造体之前完成

C++ 里可以在构造体里给 `const` 成员赋值（因为它是在进入构造体之前初始化的），Dart 里 `final` 字段必须在初始化列表或声明处就确定值，构造体里不能再赋值。

### 坑 3：没有拷贝构造——对象是引用

```dart
var a = Point(1, 2);
var b = a;      // b 和 a 指向同一个对象
// 没有自动拷贝，要手动：
var c = Point(a.x, a.y);  // 手动"拷贝"
```
